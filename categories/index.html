Go 并发编程实践

Go 语言的并发模型以 Goroutine 和 Channel 为核心，能轻松构建高性能、可扩展的服务。本篇文章带你快速上手。

Goroutine 基础

Goroutine 是 Go 的轻量级线程，可以用 go 关键字创建：

func main() {
    go worker(1)
    go worker(2)
    time.Sleep(time.Second)
}

func worker(id int) {
    fmt.Printf("worker %d\n", id)
}

Channel 通讯

Channel 提供了 Goroutine 之间安全通信的方式：

func main() {
    ch := make(chan int)

    go func() {
        for i := 0; i < 3; i++ {
            ch <- i
        }
        close(ch)
    }()

    for v := range ch {
        fmt.Println("receive", v)
    }
}

select 复用

select 可以同时监听多个 Channel：

select {
case msg := <-ch1:
    fmt.Println("来自 ch1:", msg)
case msg := <-ch2:
    fmt.Println("来自 ch2:", msg)
case <-time.After(time.Second):
    fmt.Println("timeout")
}

CI/CD 流水线实践

CI/CD 简介

CI/CD（持续集成/持续交付）是现代软件开发的核心实践，通过自动化流程提高代码质量和部署效率。

CI/CD 流程

持续集成（CI）

  • 代码提交触发自动构建
  • 运行自动化测试
  • 代码质量检查
  • 生成构建产物

持续交付（CD）

  • 自动部署到测试环境
  • 自动化测试验证
  • 部署到生产环境（可选）

GitHub Actions 示例

我们已经在这个博客中使用了 GitHub Actions 来实现自动部署。以下是一个典型的 CI/CD 流水线：

name: CI/CD Pipeline

on:
  push:
    branches: [ main ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run tests
        run: npm test
      
  build:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Build
        run: npm run build
      
  deploy:
    needs: build
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Deploy
        run: ./deploy.sh

CI/CD 最佳实践

  1. 快速反馈：保持构建和测试快速
  2. 自动化一切：减少手动操作
  3. 版本控制：所有配置都纳入版本控制
  4. 环境一致性：开发、测试、生产环境保持一致
  5. 监控和日志：记录所有操作和结果

总结

CI/CD 流水线让软件开发流程更加高效、可靠。通过自动化，我们可以更快地交付高质量的软件。

Kubernetes 入门指南

Kubernetes 简介

Kubernetes（简称 K8s）是一个开源的容器编排平台，用于自动化部署、扩展和管理容器化应用。

Kubernetes 核心组件

Pod

Pod 是 Kubernetes 中最小的部署单元，包含一个或多个容器。

Service

Service 定义了访问 Pod 的逻辑方式，提供稳定的网络端点。

Deployment

Deployment 管理 Pod 的副本和更新策略。

Namespace

Namespace 用于逻辑上隔离资源。

基本操作

创建 Deployment

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80

创建 Service

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
  - port: 80
    targetPort: 80
  type: LoadBalancer

常用命令

# 应用配置
kubectl apply -f deployment.yaml

# 查看 Pod
kubectl get pods

# 查看 Service
kubectl get services

# 查看日志
kubectl logs <pod-name>

# 删除资源
kubectl delete -f deployment.yaml

总结

Kubernetes 提供了强大的容器编排能力，让大规模容器应用的部署和管理变得简单高效。

Docker 基础教程

Docker 简介

Docker 是一个开源的应用容器引擎，让开发者可以打包应用及其依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上。

Docker 核心概念

镜像（Image）

Docker 镜像是一个只读的模板，用于创建 Docker 容器。

容器（Container）

容器是镜像的运行实例，可以被创建、启动、停止、删除。

仓库（Repository）

Docker 仓库用来保存镜像，类似于 Git 仓库。

常用 Docker 命令

# 拉取镜像
docker pull nginx

# 运行容器
docker run -d -p 80:80 nginx

# 查看运行中的容器
docker ps

# 查看所有容器
docker ps -a

# 停止容器
docker stop <container_id>

# 删除容器
docker rm <container_id>

Dockerfile 示例

FROM node:16-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "index.js"]

构建和运行

# 构建镜像
docker build -t my-app .

# 运行容器
docker run -p 3000:3000 my-app

总结

Docker 让应用的部署变得简单、一致和可重复。通过容器化，我们可以确保应用在任何环境中都能以相同的方式运行。

DevOps 入门指南

什么是 DevOps？

DevOps 是 Development（开发）和 Operations（运维）的组合词，是一种软件开发方法论，旨在缩短软件开发生命周期，提供高质量的持续交付。

DevOps 的核心原则

  1. 持续集成（CI）：频繁地将代码集成到主分支
  2. 持续交付（CD）：自动化部署流程
  3. 基础设施即代码（IaC）：使用代码管理基础设施
  4. 监控和日志：实时监控应用和系统状态
  5. 协作和沟通：开发团队和运维团队的紧密合作

DevOps 工具链

  • 版本控制：Git
  • CI/CD：Jenkins, GitHub Actions, GitLab CI
  • 容器化：Docker
  • 编排：Kubernetes
  • 配置管理：Ansible, Terraform
  • 监控：Prometheus, Grafana

为什么需要 DevOps？

  • 提高部署频率
  • 降低失败率
  • 缩短修复时间
  • 提高团队协作效率
