Docker 基础教程

Docker 简介

Docker 是一个开源的应用容器引擎，让开发者可以打包应用及其依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上。

Docker 核心概念

镜像（Image）

Docker 镜像是一个只读的模板，用于创建 Docker 容器。

容器（Container）

容器是镜像的运行实例，可以被创建、启动、停止、删除。

仓库（Repository）

Docker 仓库用来保存镜像，类似于 Git 仓库。

常用 Docker 命令

# 拉取镜像
docker pull nginx

# 运行容器
docker run -d -p 80:80 nginx

# 查看运行中的容器
docker ps

# 查看所有容器
docker ps -a

# 停止容器
docker stop <container_id>

# 删除容器
docker rm <container_id>

Dockerfile 示例

FROM node:16-alpine
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
EXPOSE 3000
CMD ["node", "index.js"]

构建和运行

# 构建镜像
docker build -t my-app .

# 运行容器
docker run -p 3000:3000 my-app

总结

Docker 让应用的部署变得简单、一致和可重复。通过容器化，我们可以确保应用在任何环境中都能以相同的方式运行。

DevOps 入门指南

什么是 DevOps？

DevOps 是 Development（开发）和 Operations（运维）的组合词，是一种软件开发方法论，旨在缩短软件开发生命周期，提供高质量的持续交付。

DevOps 的核心原则

  1. 持续集成（CI）：频繁地将代码集成到主分支
  2. 持续交付（CD）：自动化部署流程
  3. 基础设施即代码（IaC）：使用代码管理基础设施
  4. 监控和日志：实时监控应用和系统状态
  5. 协作和沟通：开发团队和运维团队的紧密合作

DevOps 工具链

  • 版本控制：Git
  • CI/CD：Jenkins, GitHub Actions, GitLab CI
  • 容器化：Docker
  • 编排：Kubernetes
  • 配置管理：Ansible, Terraform
  • 监控：Prometheus, Grafana

为什么需要 DevOps？

  • 提高部署频率
  • 降低失败率
  • 缩短修复时间
  • 提高团队协作效率
